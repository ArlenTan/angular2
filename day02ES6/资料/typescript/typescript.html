TypeScript

简介

  • TypeScript是一种由微软开发的自由和开源的编程语言，它是JavaScript的一个超集，扩展了JavaScript的语法
  • TypeScript如它的名字一样,最广为人知的特点是增加了类型系统
  • 相比弱类型语言,丢失了灵活性,但在编写大型项目时会降低程序复杂度,在编译时发现问题,减少bug增强健壮性

特性

  • 兼容ES6
  • 类型系统
  • 类
  • 接口
  • 类型注解
  • 模块

使用说明

  • typeScript无法直接在浏览器当中运行,所以使用ts需要先安装对应的编译器
  • 这个编译器的作用类似与我们之前使用less或sass编译器,它可以把ts转换为浏览器可执行的es5语法

编译器安装

  • 这里安装需要确保本地已经拥有npm包管理工具
    # 安装命令
npm install -g typescript
# 检测命令
tsc -v

  • 创建holle.ts文件,typescript编写的脚本使用ts作为后缀名,这个与less/sass也一样

类型

为什么要类型

  • 使用typeScript的类型定义有好处也有坏处
  • 下面通过一个小例子给大家演示类型的好处与弊端

javascript

  • 假如我们做一个加法运算,求两数之和
  • 如果没有类型检测,我们很有可能会不小心和一个字符串相加,造成严重后果
    let a = 10;
let b = '20';
let c = 30;
let sum = a + b + c; // 102030

typescript

  • 如果使用typescript编写上面的代码,然后进行编译
  • 你会发现详细的错误提示,这样显著提升了程序的健壮性
  • 但是缺点也显而易见,失去了js原本的简洁性,轻度影响开发效率
    let a: number = 10;
let b: string = '20';
let c: number = 30;
let sum: number = a + b + c;

基本类型

number类型

  • 数字类型的变量的只能赋值为数字,不区分整数与小数
    let num: number = 10;
num = 20;    // 编译通过
num = 'abc'; // 编译错误

string类型

  • 字符串类型的变量的只能赋值为字符或字符串
    let str: string = 'abc';
str = 'cba';    // 编译通过
str = 123;     // 编译错误

boolean类型

  • 字符串类型的变量的只能赋值为true或false
    let bol: boolean = true;
bol = false;    // 编译通过
bol = 1;         // 编译错误

undefined类型

  • undefined类型的变量只能赋值为undefined
    let undef: undefined;
undef = undefined;
undef = ''; // 编译错误

null类型

  • null类型的变量只能赋值为null
    let nu: null;
nu = null;
nu = 0; // 编译错误

补充

  • undefined与null可以赋值给其他类型
  • 用于表示未赋值或空的意思 ```typescript let num: number; let str: string; let bol: boolean;

num = undefine; str = null; bol = null;

枚举类型

简介

  • 枚举是typeScript扩展的一种数据类型,是enum来定义

作用

  • 枚举类型的数据都是有限的,可一一列出来
  • 比如一周有7天,一年有365天,性别有2种,像这样固定的数据我们就可以通过枚举来定义
  • 对于固定的数据有些人可能会想到使用const来定义,看下面的例子演示const的问题 ```javascript const gender = ['男', '女'];

gender[0]; // 正常 gender[0] = '公' // 不报错,因为const只在乎变量的引用是否被改变,并不在乎对象是否被改变 gender[2] = '人妖' // 同样不报错


#### 使用
- 上面定义了一个存储所有的性别的数组,这个性别也就是男女两种
- 像这种数据就是枚举数据,枚举数据按照常理讲是不允许改变的,但是上面我们却改变了它的值
- 为了解决这个问题,我们可以使用ts提供的`enum`关键字定义这种不可变的枚举类型的数据
```typescript
enum gender { '男', '女' };

gender[0];             // 正常
gender[0] = '公'     // 报错
gender[2] = '人妖'  // 报错

参数与返回值类型

function sum(a: number, b: number): number {
    return a + b;
}

sum(10, 20);  // 正常

参数个数

  • 在typescript中会严格检查函数调用时参数的个数 ```typescript function sum(a: number, b: number): number { return a + b; }

sum(10, 20); // 正常 sum(5); // 报错 sum(5, 8, 10); // 报错


#### 可选参数
- 在参数后面添加`?`号可以把参数设定为可选
- 或者使用`es6`语法给这个参数赋一个`默认值`
```typescript
function sum(a: number, b?: number, c: number = 50): number  {
    return b? a + b: a;
}

sum(5);                    // 正常
sum(5, 8, 10);          // 正常
sum(5, 8, 10, 20);     // 报错

任意参数与类型

  • 可以通过es6参数语法接收任意多个值,并可指定这些值的类型 ```typescript function sum(...arg: number[]): number { return arg.reduce((sum, v) => { return sum + v; }, 0); }

sum(1, 2, 3); // 正常 sum(1, 2, 3, 4, 5); // 正常 sum(1, 2, 3, 4, 'abc'); // 报错

泛型

为什么要泛型

  • 假设有一个方法,可以传入任意多个数字,组成数组并返回
    function createArray(...arg: number[]): number[] {
  return arg;
}
  • 假设我们现在要改造这个函数,让它可以实现更多需求,变得更通用
  • 改造需求是:传入的参数可以是数字,也可以是字符串等其他任意类型
  • 但是必须保证所有的参数类型一致,同时返回该类型构成的数组
    function createArray(...arg: any[]): any[] {
  return arg;
}
  • 你可能会像上面那样写,但显然,这不能保证参数的类型是一致的

使用泛型

  • 要解决上面的问题,泛型就派上用场了
  • 我们可以在函数的后面使用<>定义一个变量
  • 这个变量的值为可以描述数据类型的string/number/boolean等
  • 然后我们就可以使用这个变量来约束参数与返回值的类型
function createArray<Type>(...arg: Type[]): Type[] {
    return arg;
}

let arr1 = createArray<number>(10, 20, 30);  // 正确
let arr2 = createArray<string>('a', 'b', 'c')          // 正确
let arr3 = createArray<string>('a', 'b', 10)          // 报错

对象属性

  • 在typeScript中,无法给对象动态添加属性,必须预先定义

字面量对象

let obj = { a: 1 };
obj.a = 10;   // 正常
obj.b = 20;   // 报错

实例对象

class Person {
    name: string;
    constructor(name: string, age: number) {
        this.name = name;   // 正常,因为预先定义了name属性
        this.age = age;         // 报错,没有预先定义,无法动态添加属性
    }
}

静态属性

  • 在es6中,只允许使用static关键字定义静态方法,不能定义属性
  • 但是在ts中可以,使用static定义的属性方法,需要通过类名访问 ```typescript class Person { static maxAge: number = 250; public name: string = '匿名'; }

Person.maxAge; // 正常 new Person().maxAge; // 报错


## 访问权限修饰符
- 访问权限修饰符的作用是`限定`属性的可`访问`范围

#### 默认public
- `public`修饰的属性与方法使用`不受限`
- 默认情况下属性与方法的权限都是public
```typescript
class Person {
    public name: string;
    public age: number;
    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }
    public say(): void {
        console.log(`我是${this.name}`);
    }
}

let p = new Person('小虎', 20);
p.say();        // 正常

private

  • 只供类内部使用的属性或方法,可以通过private关键字来修饰 ```typescript class Person { private name: string; private age: number; constructor(name: string, age: number) {
      this.name = name;
  this.age = age;
    } public say(): void {
      console.log(`我是${this.name},我${this.age}岁了`);
    } }

let p = new Person('小虎', 20); p.say(); // 正常 p.name // 报错,name只能在类的内部访问 p.age // 报错,age只能在类的内部访问


#### protected
- 只供`类`或`子类`内部使用的属性或方法,可以通过protected关键字来修饰
```typescript
class Animal {
    protected name: string;
    private age: number;
    constructor(name: string, age: number) {
        this.name = name;
        this.age = age;
    }
    public say(): void {
        console.log(`我是${this.name},我${this.age}岁了`);
    }
}

class Person extends Animal {
    public sayName(): void {
        console.log(`我是${this.name}`);  // 正常
    }
    public sayAge(): void {
        console.log(`我${this.age}岁了`);  // 报错
    }
}

let p = new Person('小芳', 18);
p.say();        // 正常
p.name       // 报错,name只能在Animal与其子类中使用
p.age          // 报错,age只能在Animal类中使用

只读修饰符

readonly

  • 如果一个属性是只读的,可以添加readonly修饰符
  • 注意方法不可以使用readonly修饰符 ```typescript class Person { readonly name: string; readonly age: number; constructor(name: string, age: number) {
      this.name = name;
  this.age = age;
    } public say(): void {
      console.log(`我是${this.name},我${this.age}岁了`);
    } }

let p = new Person('小虎', 20); p.say(); // 正常 p.name = '大虎'; // 报错,name只能在类的内部访问
